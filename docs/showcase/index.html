Overview

One platform, two faces.

A premium customer storefront and a full operations back-office, sharing one config-driven, multi-tenant core. Add a salon by dropping in a config file — no code changes.

💎

Premium storefront

A conversion-focused landing page and a guided 4-step booking flow with real-time slot availability, hold-timer reservations and a waiting list.

🗂️

Complete back-office

Dashboard, lead pipeline, CRM, weekly calendar, settings, staff, rebooking automation and AI agent logs — twelve admin surfaces.

🤖

AI-assisted intake

Claude agents classify incoming leads, score confidence and escalate uncertain cases to human review — every output schema-validated.

🌍

Fully bilingual

German & English across every surface — storefront, booking, GDPR and admin — in the formal "Sie" register for the Austrian market.

🏢

Multi-tenant

Per-salon branding, services, hours and contact info injected at runtime from config. One codebase serves every tenant.

🛡️

GDPR by design

Austrian-law compliant: explicit opt-in consent, data export, deletion & anonymisation, automatic 2-year retention cleanup.

Intelligence

AI agents, kept on a leash.

  • Intake agent classifies each lead's intent and assigns a confidence score.
  • Below threshold → routed to a human-review queue, never auto-actioned.
  • Every agent output validated against a Zod schema — no raw JSON trusted.
  • Each call logged with token count, duration and estimated cost.
  • Follow-up & rebooking agents re-engage guests with consent-aware messaging.
Compliance

GDPR, the Austrian way.

  • Three explicit consent fields — no pre-checked boxes.
  • Self-service data export to JSON on request.
  • Deletion anonymises rather than hard-deletes, preserving tax records.
  • Automatic anonymisation after two years.
  • Per-tenant data-controller contact in every salon config.
